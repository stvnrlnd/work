# 1437-designer-pricing-tier-schedules

## Description

This is to create pricing grids for each designer that live in the backend. When publishing a plan and a pricing tier is designated the pricing for that pricing tier will auto populate. If items on the grid are not offered on a plan they can be manually deleted and saved or we could have check boxes in the pricing section to auto populate prices from the grid. This will speed up the publishing process and avoid human pricing errors.

When updating pricing yearly pricing we would only need to update the stored pricing grid. We would need an implementation date that would update pricing automatically. We would also want to store historical pricing to revert back to if necessary. The implementation date would add an end date to the previous pricing grid.
