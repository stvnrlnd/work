# 2777-qa-pre-review-form-elements-without-associated-labels

## Description

**Purpose:**  
Pre-review accessibility issue related to missing form labels before assigning for testing.

**Problem:**  
Some form inputs and dropdowns don’t have labels, so screen readers can’t tell what they’re for.

**Where:**

- Home
    
- Search
    
- Plan Detail (images)
    
- Cart (promo code field)
    

**Expected:**  
All inputs and selects should have a label or aria-label.

**Goal:**  
Quick pre-review to confirm scope before assigning for testing.
